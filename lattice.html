How the simulation works

A perfect matching is a way to pair every dot inside the region with exactly one partner, using only the edges that already exist. Every dot is used once.

There are usually many valid matchings. The simulation picks one and then slowly shuffles it to explore the others. Each small shuffle is called a flip.

What a flip does:

  1. Pick two matched pairs at random, say A - B and C - D.
  2. Check if the graph also contains the edges A - D and C - B.
  3. If yes, swap them: keep A - D and C - B, drop A - B and C - D.

The result is still a valid matching, just a slightly different one. Repeating this many thousands of times produces a fresh random matching from the space of all valid ones.

Speed controls how many flips are attempted per animation frame. Higher values mix faster but update less smoothly.

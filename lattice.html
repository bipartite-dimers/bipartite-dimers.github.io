How the simulation works

A perfect matching is a way to pair every dot inside the region with exactly one partner, using only the edges that already exist. Every dot is used once.

There are usually many valid matchings. The simulation picks one and then shuffles it to explore the others via Glauber dynamics with face flips.

What a face flip does:

  1. The planar graph has faces — the enclosed regions bounded by edges. Each face is a cycle of alternating black and white vertices.
  2. Pick a random face. Check whether its edges alternate between matched and unmatched around the cycle.
  3. If they do, flip the face: swap matched ↔ unmatched for every edge of that face. The result is still a valid perfect matching.

Face flips of all sizes make the Markov chain irreducible: any perfect matching can be reached from any other, so the dynamics converge to the uniform distribution over all matchings.

Steps/frame controls how many flips are attempted per animation frame (exponential scale). Higher values mix faster.
